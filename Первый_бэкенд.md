# Реализация Backend-а на Go (net/http) с PostgreSQL и Swagger

Этот проект будет представлять собой шаблон для бэкенд-приложения на Go с использованием стандартной библиотеки net/http, PostgreSQL в качестве базы данных и Swagger (OpenAPI) для документации API.

## Функциональность

-   RESTful API: Реализация основных операций (CRUD) для двух моделей данных.
-   База данных: Интеграция с PostgreSQL.
-   Документация API: Генерация документации Swagger на основе спецификации OpenAPI.

## Модели данных

Нужно реализовать две модели: User и Profile.

### User

| Поле     | Тип      | Описание                              |
| :------- | :------- | :------------------------------------ |
| user_id  | UUID     | Уникальный идентификатор пользователя |
| email    | string   | Уникальный email пользователя         |
| username | string   | Уникальное имя пользователя           |
| messages | []string | Список сообщений (текст)              |

### Profile

| Поле             | Тип      | Описание                         |
| :--------------- | :------- | :------------------------------- |
| profile_id       | UUID     | Уникальный идентификатор профиля |
| friends          | []string | Список друзей (текстовые имена)  |
| subscribes_count | int      | Количество подписок              |
| user_id          | UUID     | Внешний ключ к User              |

## Роуты API

Нужно реализовать следующие 10 роутов:

-   Users:
    -   POST /users - Создание нового пользователя
    -   GET /users - Получение всех пользователей
    -   GET /users/{id} - Получение пользователя по user_id
    -   PUT /users/{id} - Обновление пользователя по user_id
    -   DELETE /users/{id} - Удаление пользователя по user_id
-   Profiles:
    -   POST /profiles - Создание нового профиля
    -   GET /profiles - Получение всех профилей
    -   GET /profiles/{id} - Получение профиля по user_id
    -   PUT /profiles/{id} - Обновление профиля по user_id
    -   DELETE /profiles/{id} - Удаление профиля по user_id
-   Search Profile:
    -   GET /profiles/search?username={username} - Получение профиля по имени пользователя
    -   GET /profiles/search?email={email} - Получение профиля по email пользователя

## Swagger (OpenAPI)

Для документации API нужно использовать спецификацию OpenAPI (Swagger). Файл swagger.yaml должен содержать описание всех роутов, моделей данных, запросов и ответов
